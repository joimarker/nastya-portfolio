# Портфолио Насти — SMM

Одностраничный сайт-портфолио с простой админкой (логин/пароль) для
самостоятельного редактирования текста, кейсов, услуг и картинок —
без правок кода.

## Стек

- **Next.js 16** (App Router) — фронтенд и бэкенд в одном проекте
- **JSON-файл** (`data/content.json`) вместо базы данных — для сайта
  такого размера этого достаточно, и не нужна нативная компиляция
  (в отличие от sqlite3), что упрощает деплой
- **Cookie-сессия** с HMAC-подписью (Web Crypto API) — без внешних
  библиотек авторизации
- Картинки сохраняются в `public/uploads/`

## Структура

```
app/
  page.js              — главная страница (все секции сайта)
  globals.css          — дизайн-токены и стили
  admin/
    login/page.js      — страница входа
    page.js            — дашборд редактирования
    ImageUploader.js    — виджет загрузки картинок
  api/
    content/           — публичное API (GET контента для сайта)
    auth/               — login / logout
    admin/              — защищённые API для редактирования (только с сессией)
data/content.json      — весь контент сайта (редактируется через админку)
lib/
  store.js             — чтение/запись content.json
  auth.js              — подпись и проверка cookie-сессии
middleware.js           — защищает /admin и /api/admin
```

## Запуск локально

```bash
npm install
cp .env.example .env.local
# открой .env.local и задай свой пароль и секрет
npm run dev
```

Сайт: http://localhost:3000
Админка: http://localhost:3000/admin (пароль из `ADMIN_PASSWORD`)

## Переменные окружения

| Переменная | Назначение |
|---|---|
| `ADMIN_PASSWORD` | пароль для входа в `/admin` |
| `SESSION_SECRET` | случайная длинная строка для подписи сессии |

**Важно:** обязательно задать свои значения перед деплоем в продакшн —
дефолтные значения из `.env.example` небезопасны.

## Деплой на Render

1. Залить проект в GitHub-репозиторий (см. ниже)
2. На Render: **New → Web Service** → подключить репозиторий
3. Настройки сборки:
   - Build Command: `npm install && npm run build`
   - Start Command: `npm run start`
4. В разделе Environment добавить `ADMIN_PASSWORD` и `SESSION_SECRET`
5. **Важно про картинки:** Render по умолчанию не хранит файлы между
   деплоями (диск эфемерный) — загруженные через админку картинки
   пропадут при следующем деплое. Для продакшна нужно подключить
   Render **Persistent Disk** (Settings → Disks, примонтировать на
   `/opt/render/project/src/public/uploads`), иначе картинки слетят.

## Как залить в GitHub

```bash
git init
git add .
git commit -m "Первая версия сайта-портфолио"
git branch -M main
git remote add origin <ссылка-на-твой-репозиторий>
git push -u origin main
```

`.env.local` и `public/uploads/*` не попадут в git — это специально
(см. `.gitignore`), чтобы не хранить пароль и картинки в репозитории.

## Как редактировать контент

Зайти на `/admin`, ввести пароль. Можно менять:

- Главный экран (заголовок, подзаголовок, кнопка)
- Блок "Обо мне" (текст, фото, цифры/статистика)
- Кейсы — добавление, редактирование, удаление, с картинкой
- Услуги и цены — то же самое
- Контакты (Telegram, email, Instagram)

Все изменения применяются сразу после нажатия "Сохранить" — без
пересборки и передеплоя сайта.

## Дизайн-концепция

Вместо клишированного инста-градиента или дефолтных AI-палитр —
метафора **роста**: тёмно-зелёный фон + янтарный акцент (SMM — это
выращивание охватов и аудитории). Сигнатурный элемент — карточки
кейсов в hero, развёрнутые веером как "контактный лист" плёнки,
выпрямляющиеся при наведении — отсылка к закулисью создания контента,
а не к глянцевой ленте соцсети.

Шрифты: Fraunces (заголовки) + Inter (текст) + IBM Plex Mono (цифры,
теги, лейблы).
